Critères de la thématique « 11. Formulaires »

  • Chaque champ doit posséder une étiquette (ex : label).
  • Chaque label doit être associé à son champ via les attributs for et id.
  • Chaque étiquette doit être pertinente et cohérente dans l'ensemble du site.
  • L'étiquette doit être accolée au champ associé.

⚠️ L'attribut Placeholder n'est pas une étiquette.

Il peut être utilisé pour des aides à la saisie secondaires, non nécessaires à la compréhension du champ.

Il doit être identique à l'attribut title (s'il est présent).

Référence : 11.1-4
  • Les champs de même nature doivent être regroupés dans un fieldset ou un role="group".
  • Chaque groupe doit avoir une légende (ex : legend ou un titrehx avec un id relié avec aria-labelledby
  • Chaque légende doit être pertinente.
Référence : 11.5-7
  • Les items de même nature doivent être regroupés dans un optgroup label="Nom du groupe".
Référence : 11.8
  • Chaque bouton doit avoir un intitulé pertinent.
Référence : 11.9

Champs obligatoires

Un champ obligatoire doit disposer d'une indication visible et d'une indication technique qui indique sont caractère obligatoire.

Le champ doit disposer de l'attribut required.

La mention doit être visible dans l'étiquette associé.

Le caractère * n'est pas suffisant. Un texte précédent la première utilisation du symbole doit être présent.
Voir Formulaire avec étoile *

Cas particulier : Pour les formulaires ne comportant qu'un seul champ obligatoire (exemple : formulaire de recherche), les mentions ne sont pas obligatoires.

Formats ou types obligatoires

  • Si un format ou un type de donnée est requis, les instructions doivent être visible.
  • Le champ doit être relié aux instructions avec describedby="ID-MESSAGE"

Messages d'erreur

  • Les messages d'erreur doivent nommer le champ concerné.
  • Les types et formats de données doivent être suggérés si nécessaire, et un exemple doit être donné.
  • Les messages HTML5 doivent êtres personnalisés via l'API Constraint Validation.
Référence : 11.10-11

Les champ concernant des informations sur l'utilisateur doivent possèder l'attribut autocomplete avec une valeur pertinente parmi la liste des valeurs possibles.
