Critères

Référence : 11.1-4
  • Chaque champ doit posséder une étiquette (ex : label).
  • Chaque label doit être associé à son champ via les attributs for et id.
  • Chaque étiquette doit être pertinente et cohérente dans l'ensemble du site.
  • L'étiquette doit être accolée au champ associé.
Référence : 11.5-7
  • Les champs de même nature doivent être regroupés dans un fieldset ou un role="group".
  • Chaque groupe doit avoir une légende (ex : legend ou un titrehx avec un id relié avec aria-labelledby
  • Chaque légende doit être pertinente.
Référence : 11.8
  • Les items de même nature doivent être regroupés dans un optgroup label="Nom du groupe".

Formulaire accessible RGAA 4.1

Voici un exemple de formulaire implémentant les recommandations d'accessibiblité (d'autres implémentations sont possibles).

Un champ obligatoire doit disposer d'une indication visible et d'une indication technique qui indique sont caractère obligatoire.

Le champ dispose de l'attribut required.

La mention est visible dans le label associé.

Le caractère * n'est pas suffisant. Un texte précédent sa première utilisation doit être présent.

Chaque champ doit avoir une étiquette. (11.1)

<label for="field_id">Nom</label>

Les champs doivent possèder un attribut autocomplete avec une valeur pertinente parmis la liste des valeurs possibles pour l'attribut autocomplete. (11.13)
